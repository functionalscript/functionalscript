# 665-command-line-parsing-refactor. Formalize command line parsing with command maps

**Priority:** P3
**Status:** open

## Problem

Command line parsing in `fjs` uses `switch` statements that mix dispatch logic
with command metadata inline:

- `fs/fjs/module.f.ts` — top-level commands (`test`/`t`, `compile`/`c`, `cas`/`s`, `run`/`r`)
- `fs/cas/module.f.ts` — CAS subcommands (`add`, `get`, `list`)

Aliases, error messages, and handlers are scattered across `case` blocks with
no structured relationship between them. Adding a command requires a new `case`
block, and there is no way to enumerate available commands at runtime (so `help`
cannot be generated).

## Proposal

### Step 1 — Define a `CommandMap` type

```ts
type CommandEntry = {
  readonly aliases?: readonly string[]
  readonly description: string
  readonly handler: (args: readonly string[]) => Effect<NodeOp, number>
}

type CommandMap = { readonly [command: string]: CommandEntry }
```

### Step 2 — Replace `switch` blocks with maps

Build a `CommandMap` in `fs/fjs/module.f.ts` and in `fs/cas/module.f.ts` using
the type above.

### Step 3 — Generic `dispatch` helper

Write a `dispatch(map, args)` function that:
1. Looks up `args[0]` (and aliases) in the map.
2. Calls the matched handler with `args.slice(1)`.
3. Returns a structured error listing available commands when the command is
   missing or unknown.

### Step 4 — Auto-generate `help`

Because descriptions live in the map, `fjs help` / `fjs --help` can be
generated by iterating the map — no duplicated documentation.

## Files to change

| File | Change |
|---|---|
| `fs/fjs/module.f.ts` | Replace top-level `switch` with a `CommandMap` + `dispatch` |
| `fs/cas/module.f.ts` | Replace CAS `switch` with a `CommandMap` + `dispatch` |
| `fs/fjs/cli.f.ts` (new) | `CommandMap` type and `dispatch` helper |

## Benefits

- Single source of truth for aliases, descriptions, and handlers.
- `help` output derived from the map automatically.
- Consistent "unknown command X, available: …" errors produced generically.
- New commands are one map entry, not a new `case` block.
